01

What is Machine Learning?

definition · core idea · traditional vs ML
Instead of writing rules manually, you feed the machine data + answers and let it find the rules itself.
Definition

Machine Learning is a way of teaching computers to learn from data — just like we humans learn from experience. Instead of programming every step explicitly, we give machines lots of data and let them figure out patterns on their own.

Traditional Programming vs Machine Learning

Traditional Programming

  • Give rules + data → get result
  • You write all the logic manually
  • Brittle — breaks when rules change
  • Works well for well-defined problems

Machine Learning

  • Give data + result → get rules (model)
  • Machine learns patterns automatically
  • Adapts — improves with more data
  • Works well for complex, pattern-based problems
The Core Formula
Data + Algorithm → Model → Predictions. The model is the learned set of rules (weights, parameters) that can make predictions on new, unseen data.

📌 Ultra-Compact Summary

  • ML = teaching computers to learn from data without explicit programming
  • Traditional: rules + data → output | ML: data + output → rules (model)
  • ML improves with more data; traditional programming doesn't
  • Powers: Netflix recommendations, Gmail spam filter, Face ID, self-driving cars

AI vs ML vs Deep Learning

nested hierarchy · scope · examples
Think of nested circles: AI is the biggest umbrella, ML lives inside it, and Deep Learning lives inside ML.
The Hierarchy
LevelWhat It IsKey CharacteristicExamples
AI
Artificial Intelligence		 The big umbrella — science of making machines smart like humans Any system that mimics human intelligence Chess AI, Alexa, Self-driving cars
ML
Machine Learning		 Subset of AI — machines learn from data and improve over time Learns patterns without being explicitly programmed YouTube recommendations, Netflix, Gmail spam
DL
Deep Learning		 Subset of ML — uses neural networks inspired by the human brain Handles big, complex, unstructured data (images, speech, text) ChatGPT, Face recognition, Self-driving vision
Key Distinctions
AI ⊃ ML ⊃ DL. All Deep Learning is ML. All ML is AI. But not all AI is ML, and not all ML is Deep Learning.
Common mistake: People use AI, ML, and DL interchangeably. They are NOT the same. DL specifically requires neural networks. ML can work without neural networks (e.g., Linear Regression, Decision Trees).

Types of Machine Learning

supervised · unsupervised · reinforcement
Think: supervised = learning with a teacher, unsupervised = exploring on your own, reinforcement = learning by trial and error (reward/punishment).
Type 1

Supervised Learning

You give the machine labeled data — both input AND correct output. It learns to map inputs to outputs and predict on new data.

Analogy: A student learning with a teacher who gives correct answers.

Algorithms
Linear Regression Logistic Regression Decision Tree Naive Bayes SVM KNN Random Forest XGBoost
Type 2

Unsupervised Learning

You give the machine only input data — no labels. It finds patterns, groups similar items, reduces complexity, spots outliers.

Analogy: Exploring a new city without a map — discovering structure yourself.

Algorithms
K-Means DBSCAN PCA Autoencoders Isolation Forest
Type 3

Reinforcement Learning

An agent takes actions in an environment, receives rewards for good behavior and penalties for bad. Learns by trial and error to maximize cumulative reward.

Analogy: Training a dog — reward good actions, ignore/correct bad ones.

Algorithms
Q-Learning PPO DQN A3C

Supervised Learning in Detail — With Example

Machine learning is a game of prediction. Given labeled training data (input + output), the model learns a mapping function.

Income + Credit Score = input features (X). Loan = target variable (y). Model learns: high income + high credit score → Yes.

Unsupervised Learning — What It Does
Unsupervised learning doesn't predict anything. It finds hidden structure: clusters similar data (clustering), compresses data (dimensionality reduction), detects anomalies (outlier detection).

Example: Google Search groups similar search queries together. No one labeled them — the algorithm discovered the groupings on its own.

Key point: Unsupervised is often used WITH supervised learning — use unsupervised to clean/explore data, then supervised to build prediction models.

📌 Ultra-Compact Summary

  • Supervised: labeled data → predict output (classification or regression)
  • Unsupervised: unlabeled data → find structure (clustering, PCA, anomaly detection)
  • Reinforcement: agent + environment + reward signal → optimal policy
  • Supervised = most common in industry; needs expensive labeled data
  • RL = used in games, robotics, recommendation systems

Exploratory Data Analysis (EDA)

understand before modeling · 6 EDA steps · visualizations
EDA is detective work — you're interrogating your data with curiosity before touching any model or transformation.
Definition

EDA (Exploratory Data Analysis) is the process of analyzing, visualizing, and understanding your data before building any machine learning model. It tells you what your data looks like, what's wrong with it, and what patterns exist.

EDA tells you WHAT'S WRONG. Data Cleaning FIXES it. You can't clean what you don't understand.

Data Cleaning

fix what's wrong · 7 cleaning steps · decision rules
Data cleaning is fixing errors that make the data invalid. It's not glamorous, but it's 80% of the work in real-world projects.
7 Data Cleaning Steps
1
Handle Missing Values

Identify null/NaN cells. Choose strategy based on % missing and importance.

StrategyWhen to Use
Drop rows/columnsVery few missing (<5%), or column has too many missing (>70%)
Impute with MeanNumerical feature, normally distributed
Impute with MedianNumerical feature, skewed or has outliers
Impute with ModeCategorical feature
Advanced (KNN, Regression)Important feature, complex missing pattern
2
Remove Duplicates

Exact duplicate rows bias the model by counting the same observation multiple times.

df.duplicated().sum() · df.drop_duplicates(inplace=True)
3
Fix Data Types

Numbers stored as strings, dates as text — models can't learn from wrong types.

df['age'] = df['age'].astype(int) · pd.to_datetime(df['date'])
4
Handle Inconsistent Categories

Same category with different spellings/cases treated as different values by the model.

"Male", "male", "MALE" → all become "male" | df['col'].str.lower().str.strip()
5
Detect and Handle Outliers

Use IQR fence (Q1−1.5×IQR, Q3+1.5×IQR) or Z-score (|z|>3). Then decide: remove or cap.

Removing: if clearly a data error | Capping: if extreme but valid (e.g., cap to 95th percentile)
6
Fix String / Whitespace Issues

" hello " ≠ "hello" — leading/trailing spaces create false unique values.

df['col'].str.strip() · df['col'].str.replace(' ', '_')
7
Fix Logic / Domain Errors

Physically impossible values: age = -5, BMI = 150, price = -500. Replace or remove.

df[df['age'] < 0] → replace with median or drop
Remember: EDA tells you WHAT'S wrong. Data Cleaning FIXES it. These are sequential, not interchangeable steps.

📌 Ultra-Compact Summary

  • 7 cleaning steps: missing values → duplicates → data types → inconsistent categories → outliers → whitespace → domain errors
  • Missing values: drop (if few), mean/median impute (numerical), mode impute (categorical)
  • Use median imputation for skewed data — not mean (mean is pulled by outliers)
  • Always clean AFTER EDA — cleaning without understanding is guessing
  • Cleaning is ~80% of real-world data science work
07

Data Preprocessing

transform valid data into model-ready format
Data cleaning fixes errors. Preprocessing transforms valid but raw data into a format that ML algorithms can actually learn from.
Cleaning vs Preprocessing

Data Cleaning

  • Fixes mistakes, errors, inconsistencies
  • Removes invalid data
  • Makes data correct
  • Example: fixing "Male" vs "male"

Data Preprocessing

  • Transforms valid data into usable format
  • Encodes, scales, normalizes
  • Makes data model-ready
  • Example: converting "male" → 0, "female" → 1
1. Encoding Categorical Variables

ML models only understand numbers. Text categories must be converted.

MethodWhen to UseExample
Label Encoding Ordered categories (ordinal) "Low"→0, "Medium"→1, "High"→2
One-Hot Encoding Unordered categories (nominal) — no inherent rank "Red", "Blue", "Green" → 3 binary columns
Don't use Label Encoding for nominal data. If you encode "Red"=0, "Blue"=1, "Green"=2, the model might think Green > Blue > Red, which is meaningless and misleading.
from sklearn.preprocessing import LabelEncoder, OneHotEncoder
le = LabelEncoder()
df['size'] = le.fit_transform(df['size'])  # Ordinal only

# One-hot
df = pd.get_dummies(df, columns=['color'], drop_first=True)
2. Feature Transformation

Skewed distributions hurt models that assume normality. Apply mathematical transforms to make distributions more normal.

TransformUse WhenCode
Log transformRight-skewed (long right tail)np.log1p(df['col'])
Square rootModerate right skewnp.sqrt(df['col'])
SquareLeft-skeweddf['col'] ** 2
Box-CoxOptimal power transform (positive values)scipy.stats.boxcox()
3. Feature Scaling

Algorithms that use distance or gradient (KNN, SVM, Linear Regression, Neural Networks) are sensitive to feature scales. Income ($40,000) and Age (25) are on completely different scales.

MethodFormulaOutput RangeUse When
Min-Max Normalization (x − min) / (max − min) [0, 1] Known bounds, no extreme outliers
Standardization (Z-score) (x − mean) / std Mean=0, SD=1 Normal distribution, has outliers
Critical rule: Fit scaler on TRAINING data only. Apply (transform) on both train and test. NEVER fit on test data — this is called data leakage and will give falsely optimistic results.
from sklearn.preprocessing import MinMaxScaler, StandardScaler

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)   # Fit + transform on train
X_test_scaled = scaler.transform(X_test)          # Only transform on test

📌 Ultra-Compact Summary

  • Encoding: Label = ordinal only | One-Hot = nominal (no inherent order)
  • Min-Max: (x−min)/(max−min) → [0,1] | Use when no extreme outliers
  • StandardScaler: (x−mean)/std → mean=0, std=1 | Use when outliers present
  • Log transform: fixes right-skewed features (income, price, house sizes)
  • Data leakage: never fit() on test data — use fit_transform() on train, transform() on test
  • Distance-based algorithms (KNN, SVM) REQUIRE feature scaling
  • Tree-based algorithms (RF, XGBoost) do NOT require scaling

Feature Engineering

create new signals · expose hidden patterns · domain knowledge
Feature engineering is creating new features that expose useful patterns the model can't see in raw data. It's where domain expertise meets data science.
Definition & Why It Matters

Feature Engineering = creating new features or transforming existing ones to expose useful patterns that ML models can learn from. ML models don't know domain logic — you have to give them the right signals.

Better features > Better algorithms. A weak model with great features beats a strong model with poor features.
4 Common Feature Engineering Techniques
TechniqueWhat It DoesExample
Mathematical Combinations Combine existing features to create a more meaningful signal BMI = weight / height² | Price per sqft = price / area
Target-Based Flags Create binary flags based on business domain knowledge is_senior = (age > 60) | is_high_income = (income > 100k)
Binning Convert continuous values into discrete bins (groups) Age bins: [0–18], [18–35], [35–60], [60+]
Time-Based Features Extract components from datetime for temporal patterns Day of week, Month, Hour, Is weekend, Quarter, Days since event
When NOT to engineer features: Don't create features blindly. Every new feature adds noise and can cause overfitting. Validate with feature importance or correlation before keeping it.

📌 Ultra-Compact Summary

  • Feature engineering = creating NEW features from existing ones using domain knowledge
  • 4 techniques: Math combinations | Target flags | Binning | Time-based features
  • Better features > Better algorithms — feature engineering often matters more than algorithm choice
  • Every new feature adds potential noise — validate with feature importance
  • Done AFTER cleaning and BEFORE model selection
09

Feature Selection

keep what matters · remove noise · 3 methods
Feature selection is the opposite of engineering — instead of adding, you're removing irrelevant/redundant features to reduce noise and overfitting.
Why Feature Selection?
Fewer, better features → less overfitting, faster training, better accuracy, easier interpretation. Not all features are useful.
3 Categories of Feature Selection Methods
Method 1
Filter Methods

Use statistics to score each feature independently. Fast, model-agnostic.

  • Correlation matrix → remove |r| > 0.9
  • Chi-square test → categorical vs categorical target
  • ANOVA F-test → numerical feature vs categorical target
  • Mutual Information → non-linear relationships
Method 2
Wrapper Methods

Try feature subsets by actually training models. Slower but more accurate.

  • Forward Selection — add features one by one
  • Backward Elimination — start with all, remove one by one
  • Recursive Feature Elimination (RFE) — repeatedly prune
Method 3
Embedded Methods

Feature selection built into the model training process.

  • Lasso Regression → shrinks coefficients to 0
  • Random Forest → feature importance scores
  • XGBoost → gain-based feature importance
  • Ridge Regression → reduces but not zeroes coefficients
When to Use Which Method
MethodSpeedWhen to Use
FilterFast ⚡Quick baseline, high-dimensional data, early exploration
WrapperSlow 🐢Small-medium datasets, when accuracy is paramount
EmbeddedMedium ⚡⚡Tree-based models, regularized regression — best of both worlds

  • Feature selection: remove irrelevant/redundant features → less noise, less overfitting
  • Filter: statistics-based, fast (correlation, chi-square, ANOVA, MI)
  • Wrapper: model-based, slow (forward, backward, RFE)
  • Embedded: built into model (Lasso=zeroes coefs, RF/XGBoost=importance scores)
  • Engineering (add) → Selection (remove) → Modeling (train)
