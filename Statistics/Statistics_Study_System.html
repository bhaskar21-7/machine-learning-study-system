T-01

Data Tables (One-Way Table)

A table is just structured storage — rows = observations, columns = variables.

Definition

A Data Table is a tabular representation of data organized into rows (individual records) and columns (variables/features). A one-way table shows frequency counts of a single variable.

Example

FruitFrequency
Apple10
Mango6
Banana8
Every DataFrame in pandas IS a data table. df.value_counts() builds a one-way frequency table.

Bar Graph

Turn frequency counts into visible heights — compare categories at a glance.

Definition

A Bar Graph displays categorical data where each bar's height = frequency. Bars are separated (not touching).

Key Rules

AxisWhat goes on it
X-axisCategories (nominal or ordinal)
Y-axisFrequency or proportion
Do NOT use bar graph for continuous data → use histogram. Always start Y-axis at 0 to avoid misleading visuals.
Class distribution check before training. Imbalanced classes are immediately visible.

Show what percentage of the whole each category takes — proportions, not counts.

Formulas

$$\text{Angle} = \frac{f}{n} \times 360° \qquad \text{Percentage} = \frac{f}{n} \times 100$$

Solved Example

Fruits: Apple=10, Banana=8, Mango=6. Total = 24.

T-05

Measure of Central Tendency

Where is the center of the data? Three different answers depending on what "center" means.

Mean

$$\bar{x} = \frac{\sum_{i=1}^{n} x_i}{n}$$

Sum all values, divide by count. Weakness: pulled hard by outliers.

Example: [2, 4, 6, 8, 10] → Mean = 30/5 = 6

Median

Middle value after sorting. If n is even → average the two middle values.

Odd: [1,3,5,7,9] → Median = 5

Even: [1,3,5,7] → Median = (3+5)/2 = 4

Strength: Not affected by outliers (robust).

Mode

Value that appears most frequently.

Example: [2,2,6,6,8,8,9,10,10,10,10] → Mode = 10

Comparison Table

MeasureFormulaOutlier Sensitive?Best For
MeanΣx/nYES ❌Symmetric data, no outliers
MedianMiddle valueNO ✓Skewed data, has outliers
ModeMost frequentNO ✓Categorical data, finding peaks

Skewness Rule

DistributionRelationship
Symmetric (Normal)Mean = Median = Mode
Right-skewed (positive)Mean > Median > Mode
Left-skewed (negative)Mean < Median < Mode
Using mean when distribution is skewed → median is correct. For salary/income data, always report median.
Missing value imputation: mean (normal), median (skewed), mode (categorical).
SimpleImputer(strategy='mean')      # normal features
SimpleImputer(strategy='median')    # skewed features  
SimpleImputer(strategy='most_frequent')  # categorical

📌 Ultra-Compact Summary

  • Mean = Σx/n; pulled by outliers
  • Median = middle after sorting; robust
  • Mode = most frequent; for categorical
  • Right skew → Mean > Median > Mode
  • Left skew → Mean < Median < Mode
  • Use median for real-world salary/income data
T-06

Measure of Spread (Dispersion)

Central tendency tells you WHERE data is. Spread tells you HOW SCATTERED it is around that center.

Range

$$\text{Range} = x_{max} - x_{min}$$

Simple but fragile — one outlier destroys it completely.

Variance

$$\text{Population: } \sigma^2 = \frac{\sum(x_i - \mu)^2}{N} \qquad \text{Sample: } s^2 = \frac{\sum(x_i - \bar{x})^2}{n-1}$$

Why square? Prevents negative deviations from canceling positives. Why n-1 for sample? Bessel's correction — gives unbiased estimate of population variance.

Standard Deviation

$$\sigma = \sqrt{\sigma^2}$$

Average distance from the mean, in original units. This is what you report.

Example: Data = [2,4,4,4,5,5,7,9], Mean = 5 → SD = 2

IQR (Interquartile Range)

$$\text{IQR} = Q3 - Q1$$

Spread of the middle 50% of data. Robust to outliers.

Example: Q1=15, Q3=46 → IQR = 46 − 15 = 31

Comparison

MetricFormulaRobust?Unit
Rangemax − minNOSame as data
VarianceΣ(x−x̄)²/nNOSquared ⚠️
Std Dev√VarianceNOSame as data
IQRQ3 − Q1YES ✓Same as data
Population uses N, sample uses n-1. Variance units are SQUARED — always report SD, not variance.
StandardScaler uses mean + SD. RobustScaler uses IQR — better for outlier-heavy data.
from sklearn.preprocessing import RobustScaler  # IQR-based

📌 Ultra-Compact Summary

  • SD = √Variance; same units as data; use for reporting
  • IQR = Q3 − Q1; robust; use when outliers present
  • Sample variance → n-1 (Bessel's correction)
  • Box plot = min, Q1, median, Q3, max (5-number summary)
T-07

Outliers

A data point so far from the rest that it's probably noise, error, or a genuinely rare event. You need a systematic fence to find it.

IQR Fence Method (Tukey's)

$$\text{Upper Fence} = Q3 + 1.5 \times IQR \qquad \text{Lower Fence} = Q1 - 1.5 \times IQR$$

Any point outside these fences = outlier.

Step-by-Step Example

Data: [1,1,2,2,3,4,5,5,6,6,7,8,9,9,10,10,11,13,13,14,15,101]

StepCalculationResult
1. Find Q1, Q3Lower half median, upper half medianQ1=5, Q3=11.5
2. IQR11.5 − 5IQR = 6.5
3. Upper Fence11.5 + 1.5×6.521.25
4. Lower Fence5 − 1.5×6.5−4.75
5. Flag outliers101 > 21.25101 is OUTLIER ✗
Outliers ≠ always errors. In fraud detection, the outliers ARE the signal. Never blindly delete — investigate first.
Tree models (XGBoost, RF) → robust to outliers. Linear regression, KNN, SVM → sensitive. Use RobustScaler or remove.
Exam Problems

  2. Two datasets: same mean=50. Dataset A: SD=2, Dataset B: SD=20. What does this tell you?

T-08

Z-Score (Standard Score)

How many standard deviations away from the mean is this point? Converts any value to a universal comparable scale.

Formula

$$z = \frac{x - \mu}{\sigma}$$

Interpretation

Z ValueMeaning
z = 0Value IS the mean
z = +11 SD above mean (right side)
z = −11 SD below mean (left side)
|z| > 3Potential outlier
After z-scoringMean=0, SD=1

Solved Example

μ = 8, σ = 1. How far is x = 9.5 from the mean?

$$z = \frac{9.5 - 8}{1} = 1.5 \quad \Rightarrow \text{9.5 is 1.5 SDs above the mean}$$

For x = 6.5: z = (6.5−8)/1 = −1.5 → 1.5 SDs below the mean.

Z-score ≠ normalization to [0,1]. For [0,1] → use Min-Max scaling. Z-score assumes roughly normal data for percentile interpretation.
StandardScaler in sklearn applies z-score to every feature. Required before: Linear Regression, Logistic Regression, SVM, KNN, PCA.
from sklearn.preprocessing import StandardScaler
# z = (x - mean) / std
T-09

Normal Distribution & Density Curve

Smooth out a histogram infinitely → density curve. When bell-shaped and symmetric → Normal Distribution. Most important distribution in statistics.

PDF Formula

$$f(x) = \frac{1}{\sigma\sqrt{2\pi}} \, e^{-\frac{1}{2}\left(\frac{x-\mu}{\sigma}\right)^2}$$

Empirical Rule (68-95-99.7)

Range% of Data
μ ± 1σ68%
μ ± 2σ95%
μ ± 3σ99.7%

Key Properties

  • Mean = Median = Mode (perfectly symmetric)
  • Total area under curve = 1
  • Defined entirely by μ and σ
  • Asymptotically approaches 0 (never touches x-axis)

Distribution Shapes

ShapeMean vs MedianTail Direction
Symmetric (Normal)Mean = MedianBoth equal
Right Skewed (+)Mean > MedianRight
Left Skewed (−)Mean < MedianLeft
Normal ≠ all real data. Y-axis of density curve is density, NOT probability. Density at a point can exceed 1.
Central Limit Theorem: sample means → approximately normal regardless of population distribution. Naive Bayes assumes normal features.

📝 Exam Problems

  1. Heights: μ=170cm, σ=10cm. What % of people are between 160cm and 190cm?
  2. μ=100, σ=15. Find z-score for x=130. Is it an outlier?
  3. A feature has mean=500, σ=50. After StandardScaler, what is the new mean and SD?

📌 Ultra-Compact Summary

  • Z = (x−μ)/σ; positive = right, negative = left
  • Normal: bell-shaped, symmetric, mean=median=mode
  • 68-95-99.7 rule for ±1σ, ±2σ, ±3σ
  • |z| > 3 → potential outlier
  • StandardScaler = z-score transformation
T-10

Covariance

Tells you the direction of the relationship between two variables (positive or negative), but the magnitude is uninterpretable without knowing the units.

Formula

$$\text{Cov}(X, Y) = \frac{\sum_{i=1}^{n}(x_i - \bar{x})(y_i - \bar{y})}{n-1}$$

Interpretation

Cov ValueMeaning
> 0X and Y move in the same direction
< 0X and Y move in opposite directions
= 0No linear relationship
Large |Cov| does NOT mean strong relationship — could just be large-scale units. Cov = 0 does NOT mean independent (only no linear relationship).
Covariance matrix is the foundation of PCA. Highly covariant features = redundant.
np.cov(X, Y)  # 2x2 covariance matrix
T-11

Correlation

Normalized covariance. Fixes the units problem. Always in [−1, +1]. Tells you BOTH strength and direction of linear relationship.

Pearson Correlation Formula

$$r = \frac{\text{Cov}(X,Y)}{\sigma_X \cdot \sigma_Y}$$

Interpretation Scale

r ValueRelationship
+1.0Perfect positive linear
+0.7 to +0.99Strong positive
+0.3 to +0.69Moderate positive
0 to ±0.29Weak / no linear
−0.7 to −0.99Strong negative
−1.0Perfect negative linear

Covariance vs Correlation

FeatureCovarianceCorrelation
Range−∞ to +∞−1 to +1
Unit-dependent?YESNO
Tells strength?NOYES
Tells direction?YESYES
Correlation ≠ Causation. Ice cream sales and drowning rates are correlated (both rise in summer). Neither causes the other. Pearson r = 0 does NOT mean no relationship — only no linear relationship.
Feature selection: drop features where |r| > 0.9 (multicollinearity in linear models).
df.corr()                           # Pearson matrix
sns.heatmap(df.corr(), annot=True)  # Heatmap
Exam Problems

📝 Exam Problems

  1. X=[1,2,3,4,5], Y=[2,4,5,4,5]. Compute Cov(X,Y) and r.
  2. Feature A and Feature B have r=0.97. What should you do in a linear regression model?
  3. A newspaper says "ice cream sales and drowning rates have r=+0.85, so ice cream causes drowning." What's wrong?

📌 Ultra-Compact Summary

  • r = Cov/(σ_x·σ_y); range [−1, +1]
  • |r| > 0.7 = strong; |r| < 0.3 = weak
  • Correlation ≠ Causation (never forget this)
  • Pearson = linear only; Spearman = rank/monotonic
  • Covariance matrix → PCA for dimensionality reduction
T-12

Probability Basics

Probability = fraction of times an event occurs over infinite trials. Measures uncertainty.

Core Definition

$$P(A) = \frac{\text{favorable outcomes}}{\text{total outcomes}} \qquad 0 \leq P(A) \leq 1 \qquad P(S) = 1$$

Key Rules

RuleFormula
Complement\(P(A') = 1 - P(A)\)
Addition (mutually exclusive)\(P(A \cup B) = P(A) + P(B)\)
Addition (general)\(P(A \cup B) = P(A) + P(B) - P(A \cap B)\)
Multiplication (independent)\(P(A \cap B) = P(A) \times P(B)\)

Event Types

TypeDefinitionExample
Mutually ExclusiveCan't occur togetherDie shows 1 AND 2 simultaneously
IndependentOne doesn't affect the otherTwo coin flips
DependentOne affects the otherDrawing cards without replacement
Mutually exclusive ≠ independent — completely different concepts. P(A∩B) = P(A)×P(B) ONLY if A and B are independent.
T-13

Conditional Probability & Bayes' Theorem

Given that B already happened — what's the probability of A? Bayes flips this using prior knowledge to update beliefs.

Conditional Probability

$$P(A|B) = \frac{P(A \cap B)}{P(B)}$$

Bayes' Theorem

$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$
TermNameMeaning
P(A)PriorBelief about A before seeing evidence
P(B|A)LikelihoodProbability of evidence given A is true
P(A|B)PosteriorUpdated belief after seeing evidence B
P(B)MarginalNormalizing constant

Solved Example (from notes)

A = Person has Corona, B = Person was in contact with someone who has Corona

P(Corona) = 0.01, P(Contact|Corona) = 0.9, P(Contact) = 0.05

$$P(\text{Corona}|\text{Contact}) = \frac{0.9 \times 0.01}{0.05} = \frac{0.009}{0.05} = 0.18$$

Only 18% chance of having corona even after contact — because base rate P(Corona) = 0.01 is very low. This is the base rate fallacy in action.

P(A|B) ≠ P(B|A). Never confuse them. Ignoring the prior P(A) = base rate fallacy. This is one of the most common real-world statistical errors.
Naive Bayes classifier = direct application. Spam detection, medical diagnosis, text classification.
from sklearn.naive_bayes import GaussianNB

📌 Ultra-Compact Summary

  • P(A|B) = P(A∩B)/P(B) — conditional probability
  • Independent: P(A∩B) = P(A)×P(B)
  • Bayes: P(A|B) = P(B|A)×P(A) / P(B)
  • Prior × Likelihood / Evidence = Posterior
  • Naive Bayes assumes feature independence
T-14

Descriptive vs Inferential Statistics

Descriptive = summarize what you HAVE. Inferential = conclude about what you DON'T HAVE (the population) from a sample.
DimensionDescriptiveInferential
PurposeSummarize & describeMake predictions / draw conclusions
Works withActual collected dataGoes beyond data → population
Question"What does the data say?""What can we infer about the population?"
ExamplesMean, Median, SD, GraphsHypothesis Testing, CI, p-values, t-tests
CertaintyExact (computed)Probabilistic (with error margin)
Reporting a sample mean as if it's the population truth = classic descriptive→inferential error. Larger sample ≠ better if it's biased.
EDA = Descriptive statistics. A/B testing, model evaluation = Inferential statistics.
T-15

Hypothesis Testing

You have a claim. You collect data. Hypothesis testing is the formal process to decide: is your data strong enough evidence to reject the claim?

Core Framework

TermDefinition
H₀ (Null)Default assumption — no effect, no difference, status quo
H₁ (Alternative)What you're trying to prove
p-valueP(observed data or more extreme | H₀ is true)
α (significance level)Threshold for rejection, usually 0.05
$$p < \alpha \Rightarrow \text{Reject } H_0 \qquad p \geq \alpha \Rightarrow \text{Fail to Reject } H_0$$

6-Step Process

  1. State H₀ and H₁
  2. Choose α (usually 0.05)
  3. Collect sample data
  4. Compute test statistic (t, z, χ², etc.)
  5. Find p-value
  6. Decision: p < α → Reject H₀; else Fail to Reject

One-Tailed vs Two-Tailed

TypeWhen to UseH₁ Form
Right-tailedTesting if parameter > valueH₁: μ > μ₀
Left-tailedTesting if parameter < valueH₁: μ < μ₀
Two-tailedTesting if parameter ≠ valueH₁: μ ≠ μ₀
"Fail to Reject H₀" ≠ "H₀ is true." Absence of evidence ≠ evidence of absence. p-value is NOT P(H₀ is true). Low p-value = statistically significant, NOT necessarily practically significant.
A/B testing for model/feature comparison. Feature significance in regression. Checking if model improvement is real or random noise.
T-16

T-Test

When you don't know population σ, estimate it from sample (s). The t-test accounts for this extra uncertainty — especially critical with small samples.

One-Sample T-Test Formula

$$t = \frac{\bar{x} - \mu}{s / \sqrt{n}} \qquad \text{df} = n - 1$$

Types of T-Tests

TypeComparesExample
One-sampleSample mean vs known valueIs avg score different from 70?
Two-sample (independent)Two different group meansDrug A vs Drug B
PairedSame subjects, two conditionsBefore vs after treatment

Solved Example

x̄=75, μ=70, s=10, n=30

$$t = \frac{75 - 70}{10/\sqrt{30}} = \frac{5}{1.826} = 2.74$$

Critical value (df=29, α=0.05, one-tailed) ≈ 1.699. Since 2.74 > 1.699 → Reject H₀. App users do score significantly more than 70.

T-Test vs Z-Test

T-TestZ-Test
Use whenn < 30 or σ unknownn ≥ 30 and σ known
Distributiont-distributionNormal distribution
ShapeHeavier tails (more uncertainty)Normal bell curve
With tiny n (e.g., n=5), verify normality assumption first (Shapiro-Wilk test). Always state degrees of freedom. Two-sample t-test requires checking for equal variance assumption.
Testing if model accuracy improvement is statistically real or random.
from scipy import stats
t_stat, p_value = stats.ttest_1samp(sample_data, popmean=70)
if p_value < 0.05:
    print("Reject H0 — significant result")

📌 Ultra-Compact Summary

  • H₀ = null (no effect); H₁ = alternative (your claim)
  • p < α → Reject H₀; p ≥ α → Fail to Reject
  • t = (x̄ − μ) / (s/√n); df = n−1
  • T-test: unknown σ or small n; Z-test: known σ, large n
  • p-value = P(data | H₀), NOT P(H₀ | data)
  • One-tailed = directional (>, <); two-tailed = any difference (≠)
